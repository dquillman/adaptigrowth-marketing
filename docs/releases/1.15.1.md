# ExamCoach v1.15.1 Release Notes

**Release Date:** 2026-02-11
**Type:** Patch (UX fixes + internal instrumentation)

---

## Improvements

- **Usage event tracking foundation:** ExamCoach now emits lightweight usage events (`session`, `coreAction`, `completion`) to Firestore for usage scoring. Events are fire-and-forget, fail-silent, and capped to prevent spam. Session events are deduplicated to one per calendar day. Core actions are capped at 20 per session.

- **"Report a Problem" in sidebar:** Moved issue reporting from a floating action button into the left navigation menu, visible on all pages including during active quizzes. When opened during a quiz, the report automatically attaches context (question ID, quiz type, exam ID) for faster triage.

- **Explanation auto-expand on answer submission:** After submitting any quiz answer (correct or incorrect), the explanation section now auto-expands immediately. No extra click is required. The tutor breakdown is fetched for every answer, providing immediate learning reinforcement. Users can still collapse/re-expand explanations via a toggle button.

## Fixes

- **Firebase staging deploy target:** Fixed a target name mismatch between `.firebaserc` (`exam-coach-staging`) and `firebase.json` (was `staging`). Staging deploys via `firebase deploy --only hosting:exam-coach-staging` now work correctly. Deploy script updated to match.

- **Quiz footer version label:** Corrected the hardcoded version string in the quiz footer to reflect the actual app version.

## No Breaking Changes

No API changes. No authentication changes. No route changes. No subscription logic changes.

## No Schema Changes

One new Firestore collection is written to (`usage_events`) but no existing collections or document shapes are modified. The `issues` collection may now include optional context fields (`source`, `questionId`, `quizType`, `examId`) when reports are filed during quizzes.
